Release Notes

Nov 11, 2016 - v1.3.0

Highlights from PureMetrics SDK version 1.3.0 release

  • Added Order Cancellation API
  • Refactor Referrer tracking method name
  • Connection tracking at session level rather than request level

Nov 2, 2016 - v1.2.1

Highlights from PureMetrics SDK version 1.2.1 release

  • Deprecated and introduced alternate API for manual session start tracking
  • Referral tracking
  • Onboarding Measurement APIs

Nov 1, 2016 - v1.2.0

Highlights from PureMetrics SDK version 1.2.0 release

  • Custom event tracking
  • Crash Tracking
  • Deeplink Attribution
  • Device Language Tracking
  • Installer Source tracking

Sept 5, 2016 - v1.1.0

Highlights from PureMetrics SDK version 1.1.0 release

  • Supporting multiple dimensions for products.

Aug 10, 2016 - v1.0.9

Highlights from PureMetrics SDK version 1.0.9 release

  • Tracking revenue has been split into 3 steps: transaction started, success and failed. The information passed in each of the stages is different.

v1.0.8 - v1.0.7

  • Have been **Deprecated** w.r.t change in the processing and treatment of revenue. If this SDK is being used, revenue analytics cannot be done. It would work seamlessly for all engagement analytics. It is advised to move or upgrade from this SDK version.
